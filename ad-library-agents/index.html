Build notes

Where to start, what to be careful about, and how this fits the broader Pulse roadmap.

★ Build next — D14

Client Mirror Brief

Turns the platform from "competitive intelligence" into "creative ops cockpit." A daily snapshot of "you launched 4, your top 3 competitors launched 11" justifies a retainer on its own.

★ Build next — W17

Iteration Velocity Score

Benchmarking a client's tempo against the category is something nobody else sells. Most defensible Pulse upsell — requires the full 5K-ads/day pipeline to compute.

★ Build next — M06

Concept Brief Generator

Closes the loop from "we saw this winning ad" → "here is a brief your editor can shoot tomorrow." Bridge into Creator Performance Network economics.

⚠ Be careful — D12

Geo-Targeting Sniffer

Only works on EU library data. Meta exposes geo there but not in US — narrower than it looks.

⚠ Be careful — W14

Demographic Targeting Read

Inferred from visual cues, not real targeting data. Clearly label as "inferred" or it sets wrong client expectations.

⚠ Be careful — M09

Funding-Event Tie-In

Needs Crunchbase or PitchBook integration to be real — not just an ad library agent. Build cost is higher than it appears.

Decisions to make next

  1. Spec the top 3–5 — write full Pulse-style spec sheets (input, output, prompt, sample digest, recipient) for the anchor agents plus D14, W17, M06.
  2. Build sequence matrix — score all 50 on Build Effort × Client Value × Defensibility to get a clear roadmap.
  3. Pricing tier design — decide which 8–12 become the paid Pulse Intel Pro tier on top of base Pulse.
  4. Per-client vs. global — D03, D14, M01 are per-client; D15, W06, W15 are more valuable run globally across the full pipeline. House-account agents feed every client digest.
