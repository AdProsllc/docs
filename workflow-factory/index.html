The Core Insight

The factory is itself an instance of the 8-step flow — and it's self-directing.

Every workflow Ad Pros produces follows Collect → Organize → Remember → Think → Plan → Do → Review → Improve. The factory is no different. But the real breakthrough isn't just that the factory builds workflows — it's that the factory decides which workflows to build next.

When existing workflows run, the Improve Agent sees where humans kept editing briefs, which clients kept asking for new analyses, which gaps the current library can't fill. It feeds those signals to the Recommender Agent, which proposes the next workflow to build. Humans approve the queue — they don't invent it. This is what makes it autonomous: the factory's backlog is written by the factory itself.

Every workflow shipped teaches the factory to ship the next one better and to know what to ship next. That compounding loop is the whole bet.

03 · The Eight Factory Agents

Not generic meta-agents. Factory workers with specific jobs.

Each agent has a single job, a narrow output, and a clear hand-off. The Recommender opens the loop. The Improve Agent closes it. The six in between do the design work. Three gates enforce quality.

01

The Recommender Agent

NEW · Opens the loop Step 01 — Collect Fully autonomous Output: Next-workflow backlog

The most important agent — because it removes the human from the "what should we build next?" decision. Watches deployed workflow performance, client signals, Improve Agent patterns, and the Skills Library's gaps. Proposes the next workflow to add to the queue. Writes a ranked backlog of next-up workflows with justification per item.

Without this agent, the factory still needs a human to decide what to build. With it, the factory is genuinely self-directing — humans approve the queue, but agents populate it.

Reads From
Improve Agent signals (gap patterns, human-edit frequency), client request logs, workflow performance scorecards, Skills Library coverage map
Writes
Ranked backlog of proposed workflows to workflows/_backlog — each item has: proposed workflow name, evidence, expected impact score, estimated factory cost
Pattern
Evaluator-Optimizer over workflow library gaps + Agentic RAG over past requests
Autonomy
Full — proposes; humans approve what enters the active queue (fast yes/no, not original ideation)
Example

Recommender notices: (1) 3 supplement clients asked about "cohort quality by acquisition source" in the last 30 days, (2) the Improve Agent flagged that 4 workflows all had to manually add acquisition-source joins, (3) no existing workflow covers this directly. Proposes: "Acquisition Source LTV Quality" workflow — ranked #2 in next-build queue, expected impact 7.8/10, estimated factory cost $11. Human approves or deprioritizes in 10 seconds.

02

The Intake Agent

Step 01 — Collect Gate 1: Brief Input: Approved backlog item Output: Workflow Brief

Takes the approved Recommender proposal (or a direct human request) and expands it into a structured Workflow Brief. Uses the Thinking Partner Protocol V3 to interview-first if details are missing. Outputs to ClickUp as a Brief task tagged for Gate 1 review.

Reads From
Approved backlog item, Remember layer for similar past workflows, client brand sheets
Writes
Workflow Brief (goal + data sources + stage list + risk tags) to workflows/_briefs
Pattern
Prompt Chaining + Routing (branches by input format)
Autonomy
Full until Gate 1 — human approves Brief before Blueprint work begins
03

The Shape Agent

Steps 02-03 — Organize + Remember Fully autonomous Output: Canonical Shape

Picks the canonical workflow shape from the Ad Pros Shape Library — Analysis, Audit, Brief, Launch, Optimize, Retro, Investigation, Reallocation. Tags each as read-heavy (multi-agent OK, per Anthropic) or write-heavy (single-context, per Cognition). If no shape fits, flags for human — never invents silently.

Reads From
Workflow Brief, Shape Library in Remember, past workflow outcomes
Writes
Shape Selection (one of 8 canonical shapes + read/write tag + expected token budget)
Pattern
Routing + Agentic RAG over Shape Library embeddings
Autonomy
Full — selection, not a decision that moves money
04

The Stage Designer Agent (×11 parallel)

Steps 04-05 — Think + Plan Gate 2: Blueprint Parallel orchestrator-workers

The highest-leverage design agent. Eleven instances run in parallel — one per stage of the selected shape. Each designer produces its stage's atomic subtasks, agent pattern, dependencies, eval rules, and MCP servers. The Blueprint Merger joins outputs and validates the DAG. This is where LTV-template fidelity actually gets produced.

Reads From
Shape Selection, Brief, past workflows of same shape, pattern library
Writes
Stage Blueprint per stage (subtasks, agent patterns, MCP servers, evaluators, cost estimates, HITL gates)
Pattern
Orchestrator-Workers (11 parallel) + Evaluator-Optimizer for DAG validation
Autonomy
Full until Gate 2 — human approves full Blueprint before Skills work begins
05

The Skills Curator Agent

Step 03 — Remember Gate 3: Skills Library-first

Reads the Blueprint, identifies which Skills are needed. Checks the Ad Pros Skills Library first. Reuses where possible. For gaps, drafts a new Skill as a proposal — never writes to production unilaterally. Uses Claude Skills progressive disclosure: name + description always loaded (~50 tokens), SKILL.md body on trigger, scripts on demand.

Reads From
Blueprint, Skills Library (skills/*.md), industry references, client brand sheets
Writes
Skills manifest — existing Skills to reuse + new Skills proposed as full SKILL.md drafts
Pattern
Agentic RAG + NL generation with compliance-check evaluator on every new Skill
Autonomy
Full for reuse; propose-only for new Skills — human approves at Gate 3
06

The Eval Agent

Steps 06-07 — Do + Review Fully autonomous The gating function

Per Hamel Husain, Shunyu Yao, Eugene Yan: evaluation is the bottleneck, not model capability. For every workflow, generates the eval harness — test cases, assertion rules, anti-hacking scans, success metrics, cost caps. No workflow deploys without passing evals. This is what prevents the factory from shipping 100 liabilities.

Reads From
Blueprint, Skills manifest, past workflow failure modes, client constraints
Writes
Eval harness (test cases, schema checks, anti-evasion scans, cost caps) to Braintrust / Langfuse
Pattern
Evaluator-Optimizer loop + Adversarial Red Team
Autonomy
Full — evals are structural; humans review aggregate scorecards, not individual tests
07

The Deploy Agent

Step 06 — Do Fully autonomous ClickUp write access

Takes the approved Blueprint + Skills + Eval harness and actually builds the workflow in ClickUp. Creates the List, renders Custom Field values via the first-party MCP, generates the import CSV, creates parent + N subtasks with dependencies, configures Autopilot Agents, saves as template. Handles the ClickUp API's custom-fields-must-pre-exist constraint via the pre-built Field Template Pack.

Reads From
Approved Blueprint, Skills manifest, Eval harness, client workspace context
Writes
Live ClickUp List + tasks + dependencies + Autopilot Agent configs; SKILL.md files to Remember
Pattern
Prompt Chaining + Idempotent Tools (rerun-safe; no duplicate creation)
Autonomy
Full — Gates 1-3 cleared; Deploy is mechanical execution with idempotency guards
08

The Improve Agent

Step 08 — Improve Fully autonomous Closes the loop

Closes the loop back to the Recommender. Monitors every deployed workflow's actual performance — where humans kept editing briefs, which Skills got revised, which shapes underperformed. Rewrites the factory's own prompts, adds new shapes to the Library, promotes patterns into foundational playbooks. Feeds pattern signals to the Recommender so it knows what gaps to propose filling next.

Reads From
All deployed workflow logs, eval scorecards, human edit histories, client outcomes
Writes
Prompt patches to other factory agents (as PRs), new shapes to Library, gap signals → Recommender
Pattern
Reflexion + Outcome-Based Self-Improvement + DSPy/GEPA-style reflective prompt evolution
Autonomy
Propose-only for factory-agent changes; self-updates its own playbooks

05 · Gates + Operating Metrics

High human gates during design. Fewer than five during development.

The three gates are where human judgment earns its cost — everything before is parsing and retrieval, everything after is mechanical execution. Total human time per workflow: ~55 minutes, plus ~5 min/week on queue approval.

Human-in-the-Loop Policy

Three design gates — and that's it. Development is fully autonomous.

01

Brief Gate

~10 min human. Is the problem well-defined? Are we solving the right thing? Does the shape match the goal?

If no → comment to Intake Agent for revision. If yes → Blueprint design begins.

02

Blueprint Gate

~30 min human. Does the 11-stage breakdown make sense? Are HITL gates in the right places? Cost caps reasonable? Any obvious gaps?

If patches needed → specific Stage Designer re-runs that stage only.

03

Skills Gate

~15 min human. Are new Skill drafts accurate? Do they reflect real Ad Pros IP? Compliance concerns?

Existing Skills auto-approve. Only new drafts need human sign-off.

06 · The Research Foundation

Not invention. Composition.

Every piece of this factory is grounded in published research or production systems from 2024-2026. Synthesis, not speculation.

The Building Blocks

Each factory agent maps to a specific known pattern.

Outer Loop

AFlow (MetaGPT, ICLR 2025 Oral)

Monte Carlo Tree Search over code-represented workflows. Generates workflows in ~1.5 hrs that beat hand-designed ones. Inspires the Shape + Stage Designer agents.

Inner Loop

DSPy + GEPA (Stanford, 2025)

Declarative modules compiled by reflective prompt evolution. Outperforms RL using 35× fewer rollouts. Inspires the Improve Agent's reflection loop.

Signal Discovery

BabyAGI 2o (Yohei Nakajima, 2024)

The purest public instance of "agent builds agent" — dynamically creates and registers new tools at runtime. Mental model for the Recommender Agent's autonomous backlog generation.

Skills Primitive

Claude Skills (Anthropic, Oct 2025)

Progressive disclosure: name+desc always loaded, SKILL.md on trigger, scripts on demand. Open standard adopted by Microsoft, Cursor, Notion. The Skills Curator's output format.

Architecture Rule

Cognition × Anthropic (June 2025)

Read-heavy tasks benefit from multi-agent; write-heavy tasks need single shared context. Resolved by Chase June 16. Shape Agent encodes this as a tag.

Pattern Vocabulary

Anthropic Building Effective Agents

Five canonical patterns: prompt chaining, routing, parallelization, orchestrator-workers, evaluator-optimizer. What the Stage Designers emit.

Eval Discipline

Hamel Husain + Shunyu Yao (2025)

Evaluation is the bottleneck, not capability. Error analysis on real traces. Why Eval Agent gets first-class status.

Integration Surface

ClickUp MCP (Q1 2026, public beta)

First-party MCP at mcp.clickup.com/mcp. OAuth 2.1. Covers tasks, docs, comments, search. Deploy Agent's primary tool.

Self-Evolution

Voyager (NVIDIA, 2023)

Agent writes, stores, and retrieves its own skills. The mental model for how the Skills Library grows autonomously over time.

Meta-Agent Search

ADAS (ICLR 2025, Hu/Lu/Clune)

Meta-agent iteratively programs new agents in code, tests them, and archives for future discoveries. Blueprint for Improve Agent's self-rewriting.
